>

>
> 实验结果显示，RAG 将回答准确性提升了 31%。"

## 二、高频问答话术

### Q1：你的 RAG 是怎么分块的？

> "我采用了多层次分块策略：第一层，按说话人自然分段，保持语义完整性和说话人信息；第二层，使用 512 tokens 块大小 + 50 tokens 重叠的滑动窗口，避免关键信息被切分；第三层，按行动项、决策项等语义单元切分，方便结构化提取。实验显示，这种分块策略比固定长度分块的 Top-3 准确率提升了 8%。"

### Q2：时间衰减是怎么设计的？

> "我采用了指数衰减函数：decay = 0.5^(days/7)，半衰期 7 天。企业周会一周一次，一周前内容相关性降一半符合业务直觉。我跑了 3/7/14 天消融：3 天衰减太快（丢失历史信息），14 天近期优先不足，7 天最优（Top-3 准确率 85%）。"

### Q3：为什么用 GRPO 不用 DPO？

> "会议纪要的评价维度是明确的——准确性、结构化、完整性，适合用规则量化。DPO 需要大量 chosen/rejected 偏好对，GRPO 在线采样+奖励函数更高效可控。另外，我保留了 PPO 的 clip 机制防止策略跳崩，消融显示 clip ε=0.2 时训练最稳定。"

### Q4：意图识别为什么不用纯规则？

> "纯规则延迟低但准确率低，歧义场景必漏。纯小模型准确率高但延迟 200ms。我选择规则兜底——含'这场会议''刚才'直接返回（~0ms），模糊场景走 Qwen2.5-0.5B-Instruct（已加载，有@lru_cache），两者结合兼顾延迟和准确率。"

### Q5：Agentic RAG 解决了什么问题？

> "总结模式下，如果只拿最后一句做检索，会漏掉大量需要历史背景的信息。Agentic RAG 让 LLM 先读转写摘要，分析哪些议题需要补充历史背景，生成子查询列表，然后并发执行所有检索，总耗时是取 max 而不是 sum。最后把检索结果压缩到 2000 tokens 再生成。两轮 LLM 调用比一次塞入超长文本更稳定，幻觉更少。"

### Q6：项目的不足（必问）

> "1. 奖励函数是手写规则不是 learned RM——好处是可解释、迭代快，但泛化性不如 Bradley-Terry RM
> 2. Chunk 粒度太细——单句切分缺上下文，影响 Cross-Encoder 精排
> 3. 没有线上 A/B 测试，全是离线评估"

### Q7：Cross-Encoder 为什么比 Bi-Encoder 准？

> "Bi-Encoder 是独立编码后余弦相似度，token 无交互。Cross-Encoder 把 query 和 doc 拼接后过 Transformer 交叉注意力，捕捉细粒度匹配。消融：Bi-Enc Only 76% → +Reranker 82%。"

## 三、手撕代码准备

### 1. GRPO 优势计算

```python
def compute_advantage(rewards: list[float], group_size: int = 4) -> list[float]:
    advantages = []
    for i in range(0, len(rewards), group_size):
        group = rewards[i:i + group_size]
        mean = sum(group) / len(group)
        std = (sum((r - mean) ** 2 for r in group) / len(group)) ** 0.5
        std = std if std > 1e-8 else 1.0
        for r in group:
            advantages.append((r - mean) / std)
    return advantages
```

### 2. 时间衰减函数

```python
def time_decay(timestamp: str, half_life_days: float = 7.0) -> float:
    from datetime import datetime
    doc_time = datetime.fromisoformat(timestamp)
    now = datetime.now()
    days_diff = (now - doc_time).days
    decay = 0.5 ** (days_diff / half_life_days)
    return max(0.1, decay)  # 最小 0.1，避免完全忽略旧数据
```

### 3. ROUGE-L (LCS)

```python
def lcs_length(x: str, y: str) -> int:
    m, n = len(x), len(y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if x[i - 1] == y[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[m][n]

def rouge_l(gen: str, ref: str) -> float:
    lcs = lcs_length(gen, ref)
    if not gen:
        return 0.0
    return 2 * lcs / (len(gen) + len(ref))
```

### 4. PPO-clip

```python
def ppo_clip_loss(
    ratio: float,      # π_θ(y|x) / π_old(y|x)
    advantage: float,
    epsilon: float = 0.2,
) -> float:
    clipped = min(ratio, 1 + epsilon) if advantage >= 0 else max(ratio, 1 - epsilon)
    return -min(ratio * advantage, clipped * advantage)
```

## 五、Badcase 归因与解决

### Badcase 1：重复关键句刷分

- **现象**：`后端API完成80%。后端API已完成80%。后端API进度80%`
- **归因**：只有准确性和流畅度两个维度，重复关键句能同时拿两个高分
- **解决**：加结构化奖励（检查 Markdown 格式）+ 流畅度 n-gram 惩罚
- **效果**：复读率从 23% 降到 5%

### Badcase 2：编造不存在的数字（幻觉）

- **现象**：会议没讨论预算，生成"预算50万元"
- **归因**：LLM 倾向于生成具体数字来拿准确性高分
- **解决**：加忠实度维度，逐句和 RAG 文档比对
- **效果**：幻觉率从 28% 降到 18%

### Badcase 3：时间混淆（问新内容返回旧会议）

- **现象**：问"本周进度"，返回上个月会议
- **归因**：Bi-Encoder 语义上"项目进度"相似，分不出时间
- **解决**：加时间衰减 + Soft Decay
- **效果**：时间类 query 准确率从 71% 提到 84%

### Badcase 4：训练中期 reward 崩塌

- **现象**：step 400 左右，reward 从 0.65 突然掉到 0.3
- **归因**：某 batch 梯度太大，ratio 最大值到 8.0（正常 0.8-1.2）
- **解决**：clip ε=0.2 生效 + lr 从 1e-5 降到 1e-6
- **效果**：后续训练稳定收敛

## 六、Reward Hacking 防护多层机制

### 核心：KL 正则 + Hidden Evaluation + LLM-as-Judge

> "不怕维度多，怕没有监控手段 + 没有约束机制。KL 正则 + hidden evaluation 就是两层护城河。"

**1. KL 正则约束**（防跑太远）

```python
kl = compute_kl(pi_new, pi_ref)
safety_factor = 1.0 / (1.0 + kl / kl_threshold)
```

> "KL 惩罚防止 policy 偏离参考模型太远，reward hacking 只能在参考模型附近折腾。"

**2. Hidden Evaluation**（不参与梯度，监控用）

有些维度不加入 reward 计算，但额外监控，看模型有没有在 hacking 某个维度。

**3. LLM-as-Judge**（端到端评估）

定期跑完整的大模型评估，看端到端质量有没有退化。Judge 只打分，不参与梯度，防止 judge bias 被放大。

**4. 维度合并**（减少 reward interference）

```
dep + exec  →  "action_items_quality"  (0.14 变成一个)
accuracy + faithfulness  →  "content_quality"
从 9 维压到 5-6 维，signal 更集中。
```

## 七、分布式训练（可能被追问）

### DDP vs DP

```
DP（旧）：所有梯度汇总到 GPU 0 再分发，有主卡瓶颈
DDP（现在标准）：AllReduce 环形通信，无主卡瓶颈
```

### ZeRO 三个阶段

```
ZeRO-1：切分优化器状态（Adam m,v），省约 4×
ZeRO-2：再切分梯度，用 Reduce-Scatter
ZeRO-3：连参数也切分，前向/反向时 AllGather 临时拿参数

通信量：ZeRO-3 > ZeRO-2 > ZeRO-1 > 无 ZeRO
显存节省：ZeRO-3 > ZeRO-2 > ZeRO-1 > 无 ZeRO
```

### LoRA 参数

```
rank = 32, alpha = 16
实际缩放比 = alpha/rank = 0.5
可训练参数 ≈ 1.6M（占 0.3%）
```

> "0.5B 模型单卡 40GB 足够放下，不需要 ZeRO 分片。"
