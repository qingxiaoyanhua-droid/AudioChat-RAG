## 2. 核心指标

| 指标 | 数值 | 计算方法 | 测试集 |
|------|------|---------|--------|
| **转写准确率** | 92% | 1 - CER (字符错误率) | 50 段音频 |
| **Top-3 检索准确率** | 83% | 至少 1 个相关/总查询 | 200 查询 |
| **RAG 问答提升** | 31% | (0.84-0.64)/0.64 | 200 查询 |
| **GRPO 生成质量提升** | 15% | (4.1-3.6)/3.6 | 200 查询 |
| **处理时长缩短** | 30% | (100-70)/100 | 实际测量 |

### 详细计算方法

#### 转写准确率 92%
```
CER = (插入 + 删除 + 替换) / 总字符数 × 100%
准确率 = 1 - CER
测试集：50 段会议音频，人工标注正确文本
结果：CER = 8%，准确率 = 92%
```

#### Top-3 检索准确率 83%
```
Top-3 准确率 = 至少 1 个相关的查询数 / 总查询数 × 100%
测试集：200 个测试查询
结果：166/200 = 83%
```

#### RAG 提升 31%
```
无 RAG 平均分：3.2/5.0 = 0.64
有 RAG 平均分：4.2/5.0 = 0.84
提升 = (0.84-0.64)/0.64 = 31%
```

#### GRPO 提升 15%
```
SFT 平均分：3.6/5.0
GRPO 平均分：4.1/5.0
提升 = (4.1-3.6)/3.6 = 14% ≈ 15%
```

## 3. 技术架构

```
语音 → 说话人分离 → ASR → RAG 检索 → LLM → 结构化输出
                    ↓
              ChromaDB + BGE
```

### 三大核心模块

1. **SFT 监督微调** - Transformers Trainer + LoRA
2. **GRPO 强化学习** - 自己实现的 GRPOTrainer
3. **RAG 检索增强** - ChromaDB + BGE + 时间衰减

### RAG 检索流程

```
1. BGE 嵌入：query → 1024 维向量
2. ChromaDB 检索：余弦相似度，Top-15 候选
3. 时间衰减：decay = 0.5^(days/7)
4. ReRank 精排：Cross-Encoder
5. Top-3 返回给 LLM
```

## 6. 面试核心问题

### Q1: "RAG 是怎么检索的？"

**回答**:

> "RAG 检索分 5 步：
>
> 1. **BGE 嵌入**：query → 1024 维向量
> 2. **ChromaDB 检索**：余弦相似度，Top-15 候选
> 3. **时间衰减**：`decay = 0.5^(days/7)`，半衰期 7 天
> 4. **ReRank 精排**：Cross-Encoder 重新排序
> 5. **Top-3 返回**：给 LLM 生成答案
>
> **为什么用时间衰减？**
> - 会议记录时效性强
> - 一周前的进度可能已经变了
> - 让近期数据权重更高
>
> **效果**：
> - Top-3 准确率：83%
> - 问答准确性提升：31%"

---

### Q2: "GRPO 的 reward 怎么计算？"

**回答**:

> "我的 reward 函数有**5 个维度**：
>
> ```
> reward = 0.35*纪要质量 + 0.35*行动项质量 + 0.20*决策项质量 + 0.10*整体格式
> ```
>
> **1. 纪要质量（35%）**
> - 语义相似度：BGE 嵌入计算（和 reference）
> - 信息完整性：关键数字、人名覆盖率
> - 流畅度：n-gram 重复度
>
> **2. 行动项质量（35%）**
> - 召回率：reference 中的行动项有多少被提取
> - 准确率：生成的行动项有多少是正确的
> - 格式规范性：是否有负责人、任务、截止时间
>
> **3. 决策项质量（20%）**
> - 召回率 + 准确率（F1 分数）
> - 信息完整性：是否包含时间、数字等
>
> **4. 整体格式（10%）**
> - Markdown 结构
> - 章节完整性
>
> **计算流程**：
> 1. 对同一 prompt 生成 4 个 response
> 2. 计算每个 response 的 reward
> 3. 计算组内优势：`A_i = (r_i - mean(r)) / std(r)`
> 4. 计算 GRPO 损失并更新
>
> **代码位置**：`grpo_multitask_reward.py`"

---

### Q3: "为什么用 GRPO 不用 PPO？"

**回答**:

> "选择 GRPO 而不是 PPO，基于三点考虑：
>
> **1. 计算效率**
> - PPO 需要训练 Critic 模型
> - GRPO 不需要，用组内相对优势代替
> - 节省一个模型的开销
>
> **2. 实现复杂度**
> - PPO 需要调 Clip 范围、Value loss 系数等
> - GRPO 超参更少，主要是 KL 系数
> - 更容易调试
>
> **3. 稳定性**
> - PPO 的 Clip 操作有时导致梯度消失
> - GRPO 的组内归一化天然稳定
>
> **实际效果对比**：
> | 指标 | PPO | GRPO |
> |------|-----|------|
> | 训练时间 | 15 分钟 | 10 分钟 |
> | 峰值显存 | 35GB | 28GB |
> | 最终 Reward | 4.0 | 4.1 |
>
> 所以 GRPO 更适合我的场景。"

### Q4: "GRPO 的 Loss 函数是什么？"

**回答**:

> "GRPO Loss 由两部分组成：
>
> **公式**：
> ```
> L = -E[log(π_new/π_old) × A_i] + kl_coef × KL(π_new || π_ref)
>      └──────┬───────┘   └─┬─┘     └─────┬─────┘
>          策略损失     优势    KL 散度惩罚
> ```
>
> **第一项：策略损失**
> - `log(π_new/π_old)`: 新旧策略的 log 比率
> - `A_i`: 优势值（这个动作比平均水平好多少）
> - 负号：优势高的动作，要增加概率
>
> **第二项：KL 散度惩罚**
> - `KL(π_new || π_ref)`: 新模型和参考模型的差异
> - `kl_coef`: 惩罚系数（我设 0.1）
> - 作用：防止模型'学坏了'
>
> **我的实现**：
> ```python
> def grpo_loss(self, log_probs, advantages, old_log_probs, kl_div):
>     ratios = [torch.exp(lp - olp) for lp, olp in zip(log_probs, old_log_probs)]
>     policy_losses = [-r * a for r, a in zip(ratios, advantages)]
>     loss = torch.stack(policy_losses).mean() + self.args.kl_coef * kl_div
>     return loss
> ```"

## 7. GRPO Reward 计算

### 7.1 多任务 Reward 公式

```python
reward = 0.35*纪要质量 + 0.35*行动项质量 + 0.20*决策项质量 + 0.10*整体格式
```

### 7.2 计算流程

```python
# 1. 对同一 prompt 生成 4 个 response
responses = model.generate(prompt, num_generations=4)

# 2. 计算每个 response 的 reward
rewards = []
for resp in responses:
    reward, sub_rewards = reward_fn.compute_reward(
        generated=resp,
        reference=reference,
        query=query
    )
    rewards.append(reward)

# rewards = [0.75, 0.82, 0.68, 0.79]  # 4 个分数

# 3. 计算组内优势
mean_reward = np.mean(rewards)  # 0.76
std_reward = np.std(rewards) + 1e-8  # 0.05
advantages = [(r - mean_reward) / std_reward for r in rewards]
# advantages = [-0.2, 1.2, -1.6, 0.6]

# 4. 计算 GRPO 损失
loss = -E[log(π_new/π_old) * A_i] + kl_coef * KL(π_new || π_ref)
```

### 7.3 关键点

- **Reward 是样本级别的**（每个 response 一个分数）
- **不是 token 级别的**（不平均到每个 token）
- **均值方差用于计算优势**（组内归一化）
- **GRPO loss 基于优势更新策略**
