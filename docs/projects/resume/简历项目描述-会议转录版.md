## 💡 面试准备（会议转录场景）

### Q1: 会议转录系统的架构是什么？⭐⭐⭐⭐⭐

**回答**：

```
用户音频 → [语音预处理] → [ASR 识别] → [说话人分离] → [结构化纪要] → [智能问答]
              ↓              ↓            ↓              ↓              ↓
          16kHz mono    语音→文字     区分说话人     提取 action     RAG 检索
                                          (DER 15%)     items        增强生成
```

**各模块作用**：
1. **语音预处理**：统一采样率（16kHz）、降噪、VAD 切分
2. **ASR 识别**：FunASR Paraformer，语音→文字（准确率 92%）
3. **说话人分离**：3D-Speaker CAM++，区分不同说话人（DER 15%）
4. **结构化纪要**：提取关键信息（决策、action items、时间点）
5. **智能问答**：RAG 检索历史会议 + LLM 生成回答

### Q2: RAG 在会议场景怎么用？⭐⭐⭐⭐⭐

**回答**：

**会议场景的特殊需求**：
- "上周张三说了什么？"（时序查询 + 说话人过滤）
- "项目进度如何？"（跨会议检索）
- "技术决策是什么？"（关键信息提取）

**RAG 解决方案**：
```python
# 1. 会议记录存储
doc = {
    "content": "张三：后端 API 已完成 80%，预计 1 月 15 日完成",
    "meeting_id": "tech_review_2024",
    "speaker": "张三",
    "timestamp": "2024-12-15T10:00:00",
    "type": "progress"  # 进度汇报
}

# 2. 查询处理
query = "上周张三说了什么"
# → 提取：speaker="张三", time_range="上周"

# 3. 检索 + 过滤
results = chroma.query(query, speaker_filter="张三")
results = time_filter(results, time_range="上周")

# 4. 时间衰减重排序
for doc in results:
    decay = 0.5 ** (days_diff / 7.0)
    doc.score *= decay
```

**效果**：
- Top-3 检索准确率 85%
- 支持时序查询、说话人过滤
- 检索延迟 < 100ms

### Q3: GRPO 在会议场景怎么用？⭐⭐⭐⭐

**回答**：

**会议场景的特殊需求**：
- 会议纪要要准确（不能瞎编）
- 纪要要简洁（不能太啰嗦）
- 要提取关键信息（决策、action items）

**GRPO 奖励函数设计**：
```python
def compute_reward(generated_summary, reference_summary, meeting_content):
    # 1. 准确性（40%）：是否准确反映会议内容
    accuracy = cosine_sim(generated_summary, reference_summary)
    
    # 2. 流畅度（30%）：是否通顺易读
    ngrams = extract_ngrams(generated_summary, n=3)
    fluency = len(set(ngrams)) / len(ngrams)
    
    # 3. 相关性（30%）：是否聚焦关键信息
    key_points = extract_key_points(meeting_content)
    relevance = cosine_sim(generated_summary, key_points)
    
    # 加权
    reward = 0.4*accuracy + 0.3*fluency + 0.3*relevance
    return reward
```

**偏好数据构建**：
- chosen：人工标注的优质纪要
- rejected：自动生成的普通纪要
- 数据量：1000+ 样本

**效果**：
- 生成质量提升 15%
- 与人工标注相关性 0.82

### Q4: 会议转录的挑战？⭐⭐⭐

**回答**：

**挑战 1：多说话人重叠**
- 问题：开会时多人同时说话
- 解决：说话人分离（DER 15%），重叠检测与分离

**挑战 2：专业术语识别**
- 问题：技术名词、缩写识别不准
- 解决：热词增强、自定义词典

**挑战 3：长会议处理**
- 问题：1 小时 + 会议，延迟高
- 解决：流式处理（500ms 切片），边说话边转录

**挑战 4：纪要质量不稳定**
- 问题：通用 LLM 生成的纪要质量参差不齐
- 解决：GRPO 强化学习优化，提升生成质量

### Q5: 为什么选这个技术栈？⭐⭐⭐

**回答**：

**ASR：FunASR Paraformer**
- 中文识别准确率高（92%）
- 支持说话人分离
- 阿里开源，社区活跃

**RAG：ChromaDB + BGE**
- ChromaDB：轻量、API 简单、支持持久化
- BGE-Large-ZH：中文语义检索 SOTA
- 适合快速原型和中小规模部署

**RLHF：LLaMA-Factory + GRPO**
- LLaMA-Factory：支持多种 RLHF 算法
- GRPO：不需要 critic 模型，实现简单
- 支持 Qwen3-30B 等主流模型

**流式：vLLM + AsyncIO**
- vLLM：Prefix Caching，流式输出
- AsyncIO：异步并发，降低延迟

## 📋 会议转录系统核心代码（伪代码）

```python
class MeetingTranscriptionSystem:
    """会议转录系统"""
    
    def __init__(self):
        self.asr = FunASRTranscriber()
        self.diarizer = ThreeDSpeakerDiarizer()
        self.rag = AudioChatRetriever()
        self.llm = FunAudioChatLLM()
    
    def transcribe(self, audio_path: str) -> dict:
        """
        会议转录全流程
        """
        # 1. 说话人分离
        diar_segments = self.diarizer.diarize(audio_path)
        
        # 2. ASR 识别（按说话人分段）
        utterances = []
        for seg in diar_segments:
            text = self.asr.transcribe(seg.audio)
            utterances.append({
                "speaker": f"spk{seg.speaker}",
                "text": text,
                "start": seg.start,
                "end": seg.end
            })
        
        # 3. 结构化纪要
        summary = self.generate_summary(utterances)
        
        # 4. 存储到 RAG
        self.rag.add_meeting_record(
            meeting_id=extract_id(audio_path),
            utterances=utterances
        )
        
        return {
            "utterances": utterances,
            "summary": summary
        }
    
    def generate_summary(self, utterances: list) -> str:
        """生成会议纪要"""
        # 1. 格式化输入
        meeting_text = format_utterances(utterances)
        
        # 2. RAG 检索（可选，增强上下文）
        context = self.rag.retrieve("相关会议记录", k=3)
        
        # 3. LLM 生成纪要
        prompt = build_summary_prompt(meeting_text, context)
        summary = self.llm.generate(prompt)
        
        return summary
    
    def answer_question(self, query: str) -> str:
        """会议问答"""
        # 1. RAG 检索相关会议记录
        docs = self.rag.retrieve(query, k=3)
        
        # 2. LLM 基于检索结果回答
        prompt = build_qa_prompt(query, docs)
        answer = self.llm.generate(prompt)
        
        return answer
```
