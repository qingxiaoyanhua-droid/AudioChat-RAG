## 版本 1：详细版（适合 2 页简历）

---

**2025.12 - 至今 &nbsp;&nbsp; 拓数派科技有限公司 &nbsp;&nbsp; 核心研发负责人**

**项目名称：企业会议智能分析系统**

**项目概述**：面向企业会议场景的智能分析系统，输入会议音频（ASR 转文字），输出结构化会议纪要。我负责 RAG 检索增强与 LLM 后训练两个核心模块。

> 一句话定位：**我不做 ASR 模型本身，只做 ASR 结果的后处理 + LLM 生成质量优化。**

---

### 核心工作

**（1）RAG 检索增强模块（核心贡献）**
- 基于 ChromaDB + BGE-Large-ZH 实现语义检索，支持历史会议记录的跨会议关联查询
- 设计两阶段检索：Bi-Encoder 粗排 Top-15 → Cross-Encoder 精排 Top-3
- 设计时间衰减机制（指数衰减，半衰期 7 天）+ 结论性文档豁免，防止重要决策被稀释（豁免条件：含"最终决定""结论：""拍板"等句式）
- 实现说话人过滤检索，按特定发言人检索历史发言
- **效果**：内部测试集 Top-3 检索准确率 85%，RAG 将回答准确性提升 31%

**（2）LLM 后训练模块（核心贡献）**
- 基于 Qwen2.5-0.5B 实现 LoRA SFT，训练模型输出符合 Markdown 格式的会议纪要
- Response-only loss masking：prompt 部分 label=-100，只对 assistant 回复算 loss
- 设计 6 维奖励函数：准确性（30%）+ 忠实度（20%）+ 流畅度（15%）+ 相关性（15%）+ 结构化（15%）+ 长度（5%）
- 基于 GRPO 算法优化生成质量，降低幻觉率（28% → 18%）
- **效果**：生成质量提升 15%，忠实度提升 10 个点

**（3）ASR 后处理模块**
- 维护动态词表，对 ASR 转写结果进行规则纠错（200+ 映射规则）
- 处理 ASR 模型对专业术语的重复错误（同一错误持续出现）
- 对低置信度词做 mask，基于 BERT 上下文预测进行二次纠错（方案设计，未上线）

**（4）数据工程与评估**
- 设计数据生成流程，从 300 场会议构建 29,344 条 SFT 训练数据
- 构建 1,200 条 GRPO prompt，200 条分层测试集（时间类/说话人类/主题类）
- 实现多维度离线评估：RAG 检索（Top-K 准确率）+ LLM 生成（6 维奖励函数）

## 面试常见问题应对

### Q: "语音转文字是你做的吗？"

**回答**：
> "ASR 模型本身不是我做的，我用的是阿里开源的 FunASR（Paraformer-large）。
>
> 我主要做两部分：
> 1. **ASR 后处理**：维护了一个动态词表，对 ASR 的重复性错误做规则纠错（200+ 映射规则）
> 2. **ASR 结果的下游处理**：把 ASR 输出接 RAG 和 LLM 做会议总结

### Q: "那 92% 的准确率跟你有什么关系？"

**回答**：
> "92% 是 FunASR 官方在公开数据集上的指标。我也在内部 20+ 场真实会议上做了测试，准确率差不多 87%-90%。
>
> 我更关注的是：ASR 错了之后，下游的 LLM 能不能兜住。设计了忠实度奖励函数，如果生成内容和 RAG 检索结果不一致就扣分。"

### Q: "为什么选 ChromaDB 不选 FAISS？"

**回答**：
> "其实两个都可以做向量检索。ChromaDB 的优势是：
> 1. 支持元数据过滤（按会议ID、说话人、时间段过滤），我们的场景需要这些
> 2. API 更简单，适合快速迭代
>
> 后期如果数据量大了，确实可以迁移到 FAISS + MySQL 的组合，那更工业级。"

### Q: "LoRA 为什么能解决过拟合？"

**回答**：
> "我的理解是：LoRA 训练的是低秩分解矩阵（AB），更新的参数量只有全量的 0.3% 左右。本质上是在做一个**参数空间的约束**，只允许模型在特定方向上更新。
>
> 全量微调 0.5B 时测试集 loss 上升（过拟合 3-5%），LoRA 就没有这个问题。面试官您说的"带一点正则效果"我觉得是这个道理。"

### Q: "GRPO 的奖励权重怎么定的？"

**回答**：
> "两方面：
> 1. 业务优先级：准确性最重要（30%），忠实度防幻觉（20%）
> 2. 消融实验：去掉忠实度后幻觉率飙升到 35%，所以不能降权重
>
> 没有做 grid search，资源有限，是经验值 + 消融验证的折中。"

### Q: "项目上线了吗？"

**回答**：
> "目前是内部测试阶段，两三个部门在用。还没有大规模铺开。
>
> 我主要负责的是核心算法的原型验证，离线评估已经跑通了。后续如果要上线，我负责性能优化和工程化。"
