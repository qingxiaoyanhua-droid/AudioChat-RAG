# 简历指南

## 版本一：RAG+RLHF 方向（推荐，用于字节/腾讯等大模型岗）

**项目描述：**

> 在拓数派科技实习期间，我负责了大模型 RAG 检索增强和强化学习优化项目，负责从零搭建了面向企业会议场景的智能转录与分析系统 DiTX-Clerk。
>
> **技术亮点：**
>
> 1. **RAG 检索增强**：基于 ChromaDB + BGE 实现语义检索，设计了意图识别路由（规则+小模型混合），根据用户问题动态选择检索策略；引入时间衰减重排序机制（decay=0.5^{days/7}），将回答准确性从 0.64 提升到 0.84（+31%）。
>
> 2. **GRPO 强化学习**：设计 9 维奖励函数，覆盖准确性（20%）、忠实度（15%）、流畅度（16%）、依赖拓扑验证（7%）、可执行性（7%）等维度；引入 PPO-clip 机制（ε=0.2）防止策略跳崩，综合奖励从 0.51 提升到 0.78（+15%）。
>
> 3. **Agentic RAG**：实现 LLM 驱动的信息缺口分析，自动识别转写中需要补充历史背景的议题，并发执行多个子查询，结合上下文压缩生成结构化会议纪要。

## 版本二：会议转录版（用于阿里/其他厂）

**项目描述：**

> DiTX-Clerk 智能会议转录与分析系统
>
> 面向企业会议场景的多模态语音对话系统，从音频到结构化纪要的端到端自动化解决方案。
>
> - 输入：会议音频（WAV）→ 说话人分离（3D-Speaker）→ ASR 转写（FunASR，准确率 92%）→ RAG 检索增强（ChromaDB + BGE）→ LLM 生成（Fun-Audio-Chat-8B）
> - RAG 模块：设计意图识别路由（当前会议/历史会议分流）+ 时间感知检索（Soft Decay）+ Agentic Planning（信息缺口分析 + 并发检索）
> - GRPO 模块：设计 9 维奖励函数（准确性/忠实度/流畅度/可执行性等），借鉴 Chat2Workflow 的 Resolve Rate 思想

## 版本三：简洁诚实版（用于不确定性高的面试）

**项目描述：**

> DiTX-Clerk：企业会议转录与摘要系统（实习项目）
>
> - 负责 RAG 检索模块：ChromaDB + BGE 两阶段检索，加入时间衰减重排序
> - 负责 GRPO 训练：多维度奖励函数设计，PPO-clip 机制
> - 负责评估模块：ROUGE-L + BERTScore + Faithfulness 三指标体系
> - 在小样本上验证了各模块有效性（RAG Top-3 准确率 85%，GRPO 综合奖励 +15%）
