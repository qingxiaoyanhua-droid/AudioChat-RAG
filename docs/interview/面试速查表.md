## 技术面试必问 10 题

### Q1: RAG 的架构是什么？⭐⭐⭐⭐⭐

**回答**：
```
用户 Query → Query 改写 → 向量检索 → 重排序 → Top-K 文档 → LLM Prompt → 生成
```

**详细版**：
1. **Query 改写**：同义词扩展、指代消解
2. **向量检索**：BGE-Large-ZH 嵌入 → ChromaDB 检索（Top-50）
3. **重排序**：
   - Cross-Encoder（bge-reranker）重排序
   - 时间衰减：`score *= 0.5^(days/7)`
4. **生成**：Top-3 文档 + Prompt → LLM

**指标**：
- 检索延迟：< 100ms
- Top-3 准确率：85%
- NDCG@10：0.81

### Q2: GRPO 的原理是什么？⭐⭐⭐⭐⭐

**回答**：

GRPO（Group Relative Policy Optimization）是 PPO 的变体，核心思想是**用组内相对优势代替 critic 模型**。

**公式**：
```
优势计算：A_i = (r_i - mean(r)) / std(r)
损失函数：L = -E[log(π_new/π_old) * A_i] + kl_coef * KL(π_new || π_ref)
```

**流程**：
1. 对同一 prompt 生成 4 个 response（group_size=4）
2. 计算每个 response 的奖励（reward）
3. 计算组内相对优势（归一化）
4. 更新策略：最大化 log_prob * advantage

**优势**：
- 不需要 critic 模型，计算简单
- 实现容易，效果与 PPO 相当

### Q3: 奖励函数如何设计？⭐⭐⭐⭐⭐

**回答**：

八维度加权奖励（借鉴 Chat2Workflow Resolve Rate 评估思想）：

```python
reward = (0.22 * accuracy          # 准确性
         + 0.17 * faithfulness      # 忠实度（防幻觉）
         + 0.17 * fluency           # 流畅度
         + 0.10 * relevance         # 相关性
         + 0.10 * structure         # 结构化（含 owner 合规检查）
         + 0.10 * length            # 长度
         + 0.07 * dependency_reward # DAG 无环验证（对应 Resolve Rate 前置）
         + 0.07 * executability     # 任务节点可执行（对应 Resolve Rate）)
```

**六原始维度计算**：

1. **准确性（22%）**：cosine_sim(generated, reference)，归一化到 0-1
2. **忠实度（17%）**：生成内容逐句与 RAG 原文比对，低于阈值扣分（防幻觉）
3. **流畅度（17%**）：1 - ngram重复度，长度惩罚
4. **相关性（10%）**：cosine_sim(generated, query)
5. **结构化（10%**）：Markdown 标题/列表/关键词 + action_items owner 非空检查（合并了 format_reward）
6. **长度（10%）**：50-300字满分，过长过短扣分

**二新增维度（Chat2Workflow 启发）**：

1. **dependency_reward（7%）**：Kahn 拓扑排序验证 DAG 无环——Kahn 算法核心：逐个删入度为0的节点，若剩节点则判定有环。
   - 低分例子：任务0"依赖1"，任务1"依赖0"（互相等待，工作流卡死）；三角循环 0→1→2→0
   - 高分例子：0无依赖→1依赖0→2依赖1（单向链）；两个起点并行 0无依赖，1无依赖，2依赖[0,1]
2. **executability（7%）**：逐任务检查四项（各0.25分）——owner非空、task长度>=5、有时间关键词、依赖索引有效。
   - 低分例子：owner="待定"、task="干活"、deadline=""（四项全空）；depends_on=[5]但总共只有3个任务
   - 高分例子：owner:"张三" task:"完成后端API联调" deadline:"本周五" deps:[]

**为什么这个权重？**
- 准确性最重要（22%）：保证内容正确
- 忠实度+流畅度（34%）：防幻觉 + 可读性
- 两个新增维度（14%）：保证任务工作流能真正执行

**面试话术**："Chat2Workflow 发现了一个关键问题——Pass Rate（格式正确）不等于 Resolve Rate（能实际执行）。我在设计奖励函数时也遇到了同样的问题：模型生成一个格式正确的任务列表很容易，但要保证任务之间没有循环依赖、负责人非空、时间合理，却经常出错。所以我借鉴了它的两阶段评估思想，在格式奖励之外额外加了依赖验证和可执行性两个维度。"

### Q7: 遇到什么技术挑战？如何解决的？⭐⭐⭐⭐⭐

**回答**：

**挑战 1：RAG 检索效果不稳定**
- 问题：语义检索有时召回不相关文档

- 问题：训练初期 loss 震荡

  - 调小 learning rate（1e-5 → 5e-6）
  - 增加 KL 惩罚（0.1 → 0.2）
  - gradient clipping
- 结果：训练稳定，生成质量提升 15%

**挑战 3：偏好数据质量不高**
- 问题：自动构建的数据噪声大
- 解决：
  - 人工校验：标注员审核 20% 数据
  - 数据清洗：去除过短/过长/重复样本
  - 数据增强：回译、改写
- 结果：数据质量提升，人工校验准确率 95%+

### Q9: 为什么用 Cosine Similarity 而不是 Inner Product 或 L2 Distance？⭐⭐⭐

**回答**：

**三种方法对比**：
| 方法 | 公式 | 特点 |
|------|------|------|
| Cosine | `A·B / (\|A\|\|B\|)` | 关注方向，不关注 magnitude |
| Inner Product | `A·B` | 关注 magnitude + 方向 |
| L2 Distance | `\|\|A - B\|\|_2` | 几何距离，越小越近 |

**为什么用 Cosine**：
1. **BGE 向量已归一化**：归一化后 Cosine = Inner Product，无需额外处理
2. **不受向量长度影响**：语义相似度应只看方向，不看文本长度
3. **高维更稳定**：L2 Distance 在高维空间"维度灾难"，所有距离趋同

**Inner Product 的问题**：偏向长向量，高频词/长文本分数虚高

**L2 的问题**：高维稀疏检索效果差

### Q11: RAG 引入外部知识库防止 ASR 错误，如果 OOV 怎么办？⭐⭐⭐⭐

**背景问题**：
- 面试官问："要是 BERT 都没听过这个词怎么办？"

**完整回答框架**：

> "这其实是分层的：
>
> 1. **子词级别**：现代 ASR/BERT 都是子词（subword）分词，OOV 词汇可以被拆成已知子词，比如'Transformer'拆成'Trans' + 'former'，不是完全未知
>
> 2. **如果子词级别也无法处理**：确实说明训练语料和实际场景偏差大，这时我会分情况：
>    - **专有名词（产品名、人名）**：构建**领域词表**做持续预训练
>    - **领域术语**：用**DAPT（领域自适应预训练）** 增强 BERT
>    - **OOV 比例过高**：说明数据收集阶段就要引入真实会议数据
>
> 3. **根本解决**：微调 ASR 模型本身，让它学会正确识别这些词"

**DAPT 举例**：
```python
# 领域自适应预训练
model = BertForMaskedLM.from_pretrained("bert-base-chinese")
model.train(domain_meeting_corpus)  # 用会议记录语料继续预训练
```

**一句话总结**：
> "OOV 不是非黑即白的——现代模型用子词分词，真正的'完全未知'其实很少。真遇到罕见词，需要用领域数据做持续预训练或微调 ASR。"

### 1. GRPO 优势计算（必考）⭐⭐⭐⭐⭐

```python
def compute_advantage(rewards):
    """
    计算组内相对优势
    Args:
        rewards: List[float], 同一 prompt 的多个 response 的奖励
    Returns:
        advantages: List[float], 优势值
    """
    import numpy as np
    
    rewards = np.array(rewards)
    mean_reward = np.mean(rewards)
    std_reward = np.std(rewards) + 1e-8
    
    advantages = (rewards - mean_reward) / std_reward
    return advantages.tolist()

# 测试
rewards = [0.8, 0.6, 0.9, 0.7]
advantages = compute_advantage(rewards)
print(advantages)  # [0.45, -0.45, 1.34, -1.34]（近似值）
```

### 2. 奖励函数计算（必考）⭐⭐⭐⭐⭐

```python
def compute_reward(generated, reference, query, retrieved_docs, action_items=None):
    """
    计算八维度奖励（借鉴 Chat2Workflow）
    reward = (0.22*accuracy + 0.17*faithfulness + 0.17*fluency
             + 0.10*relevance + 0.10*structure + 0.10*length
             + 0.07*dep + 0.07*exec)
    """
    model = SentenceTransformer('bge-large-zh-v1.5')

    # 1. 准确性（22%）：与 reference 的 BGE 余弦相似度
    gen_emb = model.encode([generated])
    ref_emb = model.encode([reference])
    accuracy = (cosine_similarity(gen_emb, ref_emb)[0][0] + 1) / 2

    # 2. 忠实度（17%）：逐句与 RAG 原文比对，防幻觉
    faithfulness = _faithfulness_reward(generated, retrieved_docs, model)

    # 3. 流畅度（17%）：3-gram 重复度
    def extract_ngrams(text, n=3):
        chars = list(text)
        return ["".join(chars[i:i+n]) for i in range(len(chars)-n+1)]
    ngrams = extract_ngrams(generated, n=3)
    fluency = len(set(ngrams)) / len(ngrams) if ngrams else 0

    # 4. 相关性（10%）：与 query 的语义相似度
    query_emb = model.encode([query])
    relevance = (cosine_similarity(gen_emb, query_emb)[0][0] + 1) / 2

    # 5. structure（10%）：Markdown 结构 + owner 合规检查（合并了 format_reward）

    # 6. dependency_reward（7%）：Kahn 拓扑排序验证 DAG 无环（对应 Resolve Rate 前置）
    dep_r = _dependency_validity_reward(action_items)

    # 7. executability（7%）：负责人非空 + 任务具体 + 时间合理（对应 Resolve Rate）
    exec_r = _task_executability(action_items)

    reward = (0.22*accuracy + 0.17*faithfulness + 0.17*fluency
             + 0.10*relevance + 0.10*0.5 + 0.10*0.5   # structure/length 简化
             + 0.07*dep_r + 0.07*exec_r)
    return reward
```

### 3. RAG 检索（可能考）⭐⭐⭐

```python
def retrieve(query, chroma_client, embedder, k=3):
    """
    RAG 检索
    """
    from datetime import datetime
    
    # 1. 嵌入
    query_emb = embedder.encode(query)
    
    # 2. 向量检索
    results = chroma_client.query(
        query_embeddings=[query_emb.tolist()],
        n_results=k * 2  # 先取 2k 个
    )
    
    # 3. 时间衰减重排序
    docs = []
    for i, doc in enumerate(results['documents'][0]):
        meta = results['metadatas'][0][i]
        distance = results['distances'][0][i]
        
        # 时间衰减
        if 'timestamp' in meta:
            doc_time = datetime.fromisoformat(meta['timestamp'])
            days_diff = (datetime.now() - doc_time).days
            decay = 0.5 ** (days_diff / 7.0)
        else:
            decay = 1.0
        
        # 最终分数
        score = (1 - distance) * decay
        docs.append((doc, score))
    
    # 4. 排序并返回 Top-K
    docs.sort(key=lambda x: x[1], reverse=True)
    return [doc for doc, score in docs[:k]]
```
