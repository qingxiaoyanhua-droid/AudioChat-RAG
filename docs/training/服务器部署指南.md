# 服务器部署指南

## 快速部署流程

### 方式一：完整部署（推荐）

#### 1. 本地打包

在本地 Windows 上：

```bash
# 进入项目目录
cd d:\VscodeProject\Users\setupmac\DiTX-Clerk

# 压缩项目（排除不必要的文件）
# 方法 1: 使用 7-Zip 右键压缩
# 方法 2: 使用 git archive
git archive -o DiTX-Clerk.tar.gz HEAD

# 或者手动创建 zip 文件，包含以下核心文件：
# - train_sft.py
# - train_grpo.py
# - setup_rag_db.py
# - prepare_data.py
# - deploy.sh
# - requirements.txt
# - audiochat/
# - evaluation/
# - scripts/
# - docs/
```

#### 2. 上传到服务器

```bash
# 在本地 PowerShell 或 CMD 中执行
# 将压缩包上传到服务器的 /data 目录

# Windows PowerShell
scp DiTX-Clerk.tar.gz 你的用户名@10.20.126.25:/data/

# 或使用 WinSCP / FileZilla 图形化工具
# 服务器地址：10.20.126.25
# 用户名：你的用户名
# 密码：你设置的密码
# 上传路径：/data/DiTX-Clerk/
```

#### 3. 登录服务器并解压

```bash
# 登录服务器
ssh 你的用户名@10.20.126.25

# 进入/data 目录
cd /data

# 创建项目目录
mkdir -p DiTX-Clerk
cd DiTX-Clerk

# 解压项目
tar -xzf /data/DiTX-Clerk.tar.gz -C . --strip-components=1

# 或者如果是 zip 文件
# unzip /data/DiTX-Clerk.zip -d .
```

#### 4. 激活环境并运行部署

```bash
# 激活 3D-Speaker 环境
source /home/ditx/llm/voice/3D-Speaker/.3ds/bin/activate

# 或者直接使用该环境的 Python
PYTHON="/home/ditx/llm/voice/3D-Speaker/.3ds/bin/python3"

# 运行部署脚本
chmod +x deploy.sh
./deploy.sh
```

## 完整训练流程

### 步骤 1: 准备训练数据

```bash
PYTHON="/home/ditx/llm/voice/3D-Speaker/.3ds/bin/python3"

# 生成 SFT 和 GRPO 训练数据
$PYTHON prepare_data.py \
    --output_dir data \
    --num_sft 1000 \
    --num_grpo 1000 \
    --num_rag 50
```

### 步骤 2: 搭建 RAG 数据库

```bash
# 使用生成的会议数据搭建 RAG
$PYTHON setup_rag_db.py \
    --storage_dir ./rag_storage \
    --data_dir data/meeting_records \
    --mode create
```

### 步骤 3: SFT 监督微调训练

```bash
# 单卡训练（调试用）
$PYTHON train_sft.py \
    --model_name_or_path /data/models/Voice/FunAudioLLM/Fun-Audio-Chat-8B \
    --train_data data/sft_train_data.jsonl \
    --output_dir saves/sft_model \
    --per_device_train_batch_size 2 \
    --gradient_accumulation_steps 4 \
    --num_train_epochs 3 \
    --learning_rate 1e-5 \
    --use_lora True \
    --lora_r 64 \
    --lora_alpha 16

# 多卡训练（8 卡 A100）
torchrun --nproc_per_node=8 train_sft.py \
    --model_name_or_path /data/models/Voice/FunAudioLLM/Fun-Audio-Chat-8B \
    --train_data data/sft_train_data.jsonl \
    --output_dir saves/sft_model \
    --per_device_train_batch_size 2 \
    --gradient_accumulation_steps 4 \
    --num_train_epochs 3 \
    --learning_rate 1e-5 \
    --use_lora True
```

### 步骤 4: GRPO 强化学习训练

```bash
# 多卡训练（4 卡 A100）
torchrun --nproc_per_node=4 train_grpo.py \
    --model_name_or_path saves/sft_model \
    --train_data data/grpo_train_data.jsonl \
    --output_dir saves/grpo_model \
    --num_generations 4 \
    --per_device_train_batch_size 2 \
    --gradient_accumulation_steps 4 \
    --num_train_epochs 3 \
    --learning_rate 1e-6 \
    --kl_coef 0.1
```

### 步骤 5: Pipeline 测试

```bash
# 运行完整 Pipeline 测试
$PYTHON scripts/offline_pipeline.py \
    --audio examples/test.wav \
    --funasr-model /data/models/Voice/iic/speech_paraformer-large-vad-punc_asr_nat-zh-cn-16k-common-vocab8404-pytorch \
    --funaudiochat-model saves/grpo_model \
    --enable-rag \
    --meeting-id test_001 \
    --output-dir output
```

## 常见问题

### Q1: SSH 登录失败

**错误信息**: `WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!`

**解决方案**:
```bash
# 清除旧的 SSH 密钥
ssh-keygen -R 10.20.126.25

# 重新登录
ssh 你的用户名@10.20.126.25
```

---

### Q2: 权限不足

**错误信息**: `Permission denied`

**解决方案**:
```bash
# 确保在/data 目录下操作
cd /data

# 不要使用 sudo（除非必要）
# 如果必须，联系管理员
```

---

### Q3: 显存不足

**解决方案**:
```bash
# 减小 batch size
--per_device_train_batch_size 1

# 增加梯度累积
--gradient_accumulation_steps 8

# 启用梯度检查点
--gradient_checkpointing True
```

---

### Q4: 依赖安装失败

**解决方案**:
```bash
# 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 或逐个安装
pip install torch transformers datasets accelerate peft
pip install chromadb sentence-transformers scikit-learn
```

---

### Q5: 模型路径不存在

**解决方案**:
```bash
# 检查模型目录
ls /data/models/Voice/

# 如果模型不存在，需要联系管理员下载
# 或修改代码中的模型路径指向你自己的模型
```
